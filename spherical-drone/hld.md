# Spherical Drone + Dock Station — High-Level Design

> スコープ: **機構・筐体・部品（CAD で作る物理設計）**。
> 電気・ソフトウェア設計は [design.md](./design.md) を前提として参照する（重量予算 55–99 g、
> 開口球形ケージ推奨、接点式充電推奨、Crazyflie 2.1 Brushless を参照実装とする、など）。

## Problem

手のひらサイズの室内ドローンと、その自動給電ステーションの**3Dプリント可能な構造部品一式**を設計する。

| R2 | 外形は手のひらサイズ = 球径 120–160 mm | design.md §目標スペック |
| R3 | コントローラは Raspberry Pi Pico W（51 × 21 mm、穴ピッチ 47 × 11.4 mm）を搭載 | ユーザー要求 |

| R5 | 印刷部品の合計重量はフレーム+ガードで **≤ 25 g**（機体総重量 60–75 g 成立ラインのため） | design.md §重量予算 |
| R6 | ステーション着陸時に機械的に自己位置決めされ、ヨー角任意のまま充電接点が導通する | design.md §7（接点式充電） |

## Approach

**クアッドロータ + 開口球形ケージ**（design.md §3 の推奨案）。中心に十字フレーム（センターデッキ）、
赤道に構造リング、上下に測地線（ジオデシック）パターンの半球ケージを被せる 3 ピース構成。
球の下極には**着陸フット**（同心円の充電接点リングを持つ円盤）が一体化され、ステーション側の
すり鉢（ファンネル）に球が自重で滑り込むことで XY が自己センタリング、同心円接点でヨー角が
任意のまま導通する。

```
   ドローン断面（正面から）                    寸法の根拠
        ___------___
      /   cage-top   \           球外径 OD = 150 mm（R2 の中央値）
     /  ___________   \          ストラット Ø3、内半径 ≈ 72 mm
    | =[deck+arms ]= ◄|──赤道リング（デッキ一体、ケージ取付座）
    |  motor│ │motor  |          プロペラ面はデッキ上面
     \  bat─┴─┘      /           モータピッチ（対角)80 mm、40mmプロペラで
      \___       ___/             先端掃引半径 = 40+20 = 60 mm → ケージまで 12 mm
          ‾‾|‾|‾‾
            └─┘◄── 着陸フット Ø36（充電リング2重 + ToF窓）

   デッキ上面図                          ステーション断面（1ポート分）
      M1   M2                          \                       /
       \   /                            \      45° cone       /
        \ /        Pico W 51×21 を        \                 /
     [=deck=]      中央に横置き、           \    ___O___    /  ← 球が円錐に線接触で
        / \        アーム4本が斜めに放射      \__|pogo |__/      自己センタリング
       /   \                                  [pin base]
      M4   M3      モータ対角ピッチ 80 mm      ファンネル開口 Ø170、底穴 Ø40
```

### ドローン（印刷部品 4 点 + 接点ディスク）

1. **center-deck** — 十字フレーム。Pico W を上面に横置き（M2 ボス 47×11.4）、ESC・IMU を下面、
   アーム 4 本の先端にモータマウント（Ø8.6 座 + M1.4×3）。赤道リングと一体で、リング上下に
   ケージ取付タブ（M2 ×4 ずつ）。
2. **cage-top** — 上半球。ジオデシックストラット Ø3 mm。頂部に Ø50 の開口（電池交換・USB アクセス用ハッチ穴を兼ねる）。
3. **cage-bottom** — 下半球。下極に着陸フット（Ø36 円盤）が一体。フット中央に ToF 窓は置けないため
   フット脇 15 mm オフセットに Ø8 ToF 窓を開ける。
4. **battery-tray** — デッキ下面にクリップ留めする電池ホルダ（重心を下げる）。
5. **contact-disc**（非印刷・別途製作）— Ø36 の片面 PCB または銅箔ディスク。中心パッド Ø8（＋極）と
   リングパッド Ø20–26（−極）。フットに接着または M2 ×2 で固定。

### ステーション（モジュール 3 分割 + 小物）

K1 Max のベッド（300×300）に収めるため、**[ポートA][コア][ポートB] の 3 モジュールを M4 ボルトで連結**。
全体外形 ≈ 480 × 200 × 90 mm。

1. **port-module ×2** — すり鉢ファンネル（開口 Ø170、円錐 45°、底穴 Ø40）。底穴の下に pogo pin
   ベース（中心 1 本 + 半径 11.5 mm の円周上に 3 本、ストローク 2–3 mm 品）。外壁に AprilTag
   プレート差し込みスロット。
2. **core-module** — Jetson Orin Nano Super Dev Kit ベイ。前面吸気スリット、上面排気、背面に
   電源・USB・LAN の切り欠き。天面に AprilTag、カメラマスト取付ボス（M4、将来の俯瞰カメラ用）。
3. **pogo-base insert** — pin ホルダはポートと別体（接点まわりだけ交換・調整できるように）。
4. **apriltag-plate** — 平板にタグ印刷シールを貼る台座。各ポート 1 + コア 1。

### 充電シーケンスと機構の対応

着陸進入（AprilTag 誘導, Jetson 側）→ 球がファンネルに接地 → 自重で円錐を滑り中心へ →
フットが底穴に落ち pogo pin がリングを押す（pin ストロークで公差吸収）→ ドローンが接点電圧を
検出して disarm + 充電開始（R10）→ 満充電後、その場から垂直離陸（ファンネルは離陸方向に開いている）。

## Alternatives

### ケージ構造

| 案 | 説明 | トレードオフ |
|----|------|--------------|
| **A. ジオデシック半球 ×2（推奨）** | 三角形格子のストラット球。赤道リングにネジ留め | 剛性/重量比が最良。モデリングは球面頂点生成が必要でやや複雑。FDM では下半球の印刷向きに工夫が要る |
| B. 緯線リング + 経線リブ（地球儀型） | 水平リング 4–5 本を縦リブで結ぶ | モデリングが簡単で印刷向きも素直。同剛性なら A 比 +20–30% 重く、極付近の開口が大きく指が入りやすい |
| C. ロータごとにダクト + 球形フレーム | design.md §3 の第2案 | 安全性最高だが +15–25 g で R5 を超過し、飛行時間が破綻する。却下 |

### 充電接点方式

| 案 | 説明 | トレードオフ |
|----|------|--------------|
| **A. 同心円接点 + pogo pin（推奨）** | フット側にリングパッド、ステーション側にバネピン | ヨー角フリー（R6 充足）。リングディスクの別途製作が要る |
| B. セグメントパッド + ヨーキー | ファンネルにキー溝を付けヨーも機械拘束 | 接点は単純だが、着陸時にヨーまで合わせる必要があり誘導要件が跳ね上がる |
| C. Qi 無線給電 | コイルを機体・ステーション双方に搭載 | 機体 +10 g 超・効率 50% 台。design.md §目標スペックで却下済み |

### ステーション構成

| 案 | 説明 | トレードオフ |
|----|------|--------------|
| **A. 3 モジュール直列連結（推奨）** | [ポート][Jetson コア][ポート] をボルト連結 | 各部品が 300mm ベッドに収まる。ポート間 300 mm 超で R8 充足。連結部の設計が増える |
| B. 一体型筐体 | 1 プリントで全体 | 480 mm 幅はベッド超過で印刷不能。却下 |
| C. ポート別置き（ケーブル接続） | コアとポートを完全分離し配線で繋ぐ | 配置自由度は最高だが、AprilTag とポートの相対位置が固定されず誘導キャリブレーションが毎回必要 |

### 製造プロセス

ユーザー要求（R4）で FDM 指定。材料は design.md 推奨に従い **PETG を第一候補**とする。
ただし CF 入りは Wi-Fi アンテナ遮蔽の懸念があり機体には使わない（Concerns #10）。
ケージのみ、割れ対策として PA12（外注 SLS/MJF）への置換余地を残すパラメトリック設計にする。

## Usage Guide

### 組み立て（ドローン）

1. center-deck 下面に 4-in-1 ESC を M2 で留め、モータ 4 個をアーム先端に M1.4 ×3 で固定、リード線をアーム溝に這わせて ESC へ半田付け
2. プロペラをモータ軸に圧入（CW/CCW の向きに注意）
3. デッキ上面に Pico W を M2 ×4 で固定、IMU/気圧基板をデッキ中央下面に、ToF を下向き窓位置に固定
4. battery-tray に LiPo を入れ、デッキ下面にクリップ留め。⚠️ 電池リードと充電基板の置き場が ESC と干渉しないか LLD で配線スペースを確認
5. contact-disc を cage-bottom のフットに M2 ×2 で固定し、リード線をストラット沿いにデッキまで這わせる ⚠️ ストラット沿い配線は見た目・空力・断線リスクあり。下極→デッキ間に専用配線チャネル付きストラットを 1 本立てるべき
6. cage-bottom を赤道リング下面に M2 ×4 で留める
7. cage-top を赤道リング上面に M2 ×4 で留める。⚠️ 以後、電池交換は頂部 Ø50 開口から手を入れて行う — トレイのクリップが指 1 本で外せる形状か要検証（Concerns #11）

### 組み立て（ステーション）

1. core-module に Jetson Dev Kit を置き M3 で固定、背面切り欠きから電源・LAN を引き出す
2. pogo-base insert に pogo pin 4 本を圧入し、リード線を半田付けして port-module 底穴下に M2 で固定
3. port-module ×2 を core-module 側面に M4 ×4 で連結
4. 充電電源（電流制限付き 5 V）を core 内の分配基板から各 pogo-base へ配線
5. apriltag-plate にタグシールを貼り、各ポートとコア天面のスロットへ差し込む

### 運用サイクル

1. ドローンがポート上空 ≈ 300 mm で AprilTag を捕捉しつつ降下
2. 球がファンネル斜面に接地 → 自重で滑って中心へ（ここから誘導精度は不要。許容着陸誤差 = 開口半径 85 − 球半径 75 ≈ **XY ±50 mm 程度**）⚠️ 斜面で跳ねた場合に隣ポートへ転がり出ないか — 開口縁の高さで抑える
3. フットが底穴に落ち、pogo pin が接点を押し上げる → 接点電圧検出 → disarm → 充電
4. 満充電 → ファンネル内から垂直離陸 ⚠️ 円錐内の吹き返し（ground effect）で離陸直後の姿勢が乱れる可能性 → 全速で一気に抜ける運用とし、円錐は 45° より浅くしない

## Concerns

1. **ケージ重量と R5**: OD150・Ø3 ストラットのジオデシック半球 2 つ + フットで概算 18–24 g（PETG, 充填100%梁）。デッキ 8–10 g と合わせると R5 の 25 g を**超過する見込み**。ストラット Ø2.5 化・分割数削減・トラス断面化のどれで削るか LLD で質量計算が必要
2. **プロペラ–ケージクリアランス 12 mm vs ケージ剛性**: Ø3 PETG ストラットの赤道付近を指で 5 N 押した時の変形が 12 mm を超えると接触事故になる。三角格子なら数 mm 程度と見込むが、根拠は試し刷り + 実測しかない
3. **下半球の FDM 印刷向き**: ドーム形状はどの向きでもストラットがオーバーハングになる。開口面を下にして印刷すると赤道付近のストラットが 45° 超の張り出しになる箇所が出る。サポート付きで印刷して表面品質が許容か、または半球をさらに 2 分割（オレンジの皮型 4 分割）して平置き印刷するか、テスト印刷で決める
4. **推力余裕**: AUW 70 g 想定で必要推力 ≥ 140 g（推重比 2）。0802 + 40 mm プロペラは 1 基 25–30 g が相場で計 100–120 g と**不足気味**。51 mm（2"）プロペラに上げるならモータピッチを 80→70 mm に詰めるか OD を 160 mm に上げるかの再配置が必要。**プロペラ径とモータ品番が確定しないと deck と cage の寸法が凍結できない**
5. **接点の導通信頼性**: 印刷公差 ±0.3 mm + 球の着座姿勢ばらつきで、フット高さが pin ストローク（2–3 mm）の範囲に収まるかが成立条件。フット垂下量とpin 突き出し量の公差スタックを LLD で計算する
6. **ToF 窓のオフセット**: フット脇 15 mm オフセットだと、機体が傾いた時の高度誤差は ≈ 15·sin(傾き)。10° で 2.6 mm — 許容範囲だが、着陸最終段の高度は ToF でなく接点検出に委ねる前提とする
7. **球の着座姿勢**: 下半球のストラットの「節」が円錐に当たると、接触円が歪んで数 mm 偏心しうる。下半球の接触円が通る緯度（cos45° 帯）には連続リングを 1 本入れてストラット節を接触帯から外す
8. **ファンネル内の吹き返し**: 着陸最終 50 mm で円錐に囲まれ、推力が 10–20% 増減する乱れが出る見込み。機構側でできるのは円錐底部に通風穴を開けることくらい。底穴 Ø40 + 通風スリットを入れる
9. **Jetson 熱設計**: Dev Kit は最大 25 W。コアモジュールは前面吸気→上面排気の自然対流 + Dev Kit 純正ファン排気を塞がない配置にする。密閉したら確実に熱だれする
10. **CF フィラメントの RF 遮蔽**: design.md は CF 入りを推奨候補に挙げるが、導電性 CF が Pico W の Wi-Fi アンテナを取り囲むと通信距離が大きく落ちる。**機体は非導電材（PETG/PA12）に限定**する
11. **電池交換アクセス**: 飛行 4–6 分/充電 30–60 分の比率だと開発中は手動電池交換が頻発する。頂部 Ø50 開口で足りるか、cage-top の一部をスナップ式ハッチにするか
12. **モータ取付穴の現物公差**: 0802/1102 はメーカーごとに取付穴仕様が違う（M1.4×3 ピッチ円 Ø6.6 は RCINPOWER 系の例）。品番確定までモータマウントはパラメータ化しておく
13. **Jetson Dev Kit の固定穴**: キャリアボードの取付穴位置は公式 3D データから実測が要る。HLD 時点ではゴム足置き + 周囲リブ拘束を仮置き
