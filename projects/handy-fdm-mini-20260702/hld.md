# Handy FDM Mini — High-Level Design

片手で持ち運べる小型 FDM 3D プリンタ。ラボ・出張先・机上で治具、スペーサ、ケース部品、センサマウントなどの小物部品を短時間で出力する携帯機。大型機の代替ではない。

初版構想（ESP32 単体＋独自ファーム、ヘッド XY＋ベッド Z 昇降、Bowden）に対するレビュー
（Claude + Codex クロスレビュー、2026-07-02）を反映した修正版仕様。

## Problem

### 要求

| 項目 | 要求 |
|---|---|
| 造形範囲 | 80 × 80 × 80 mm |
| 本体サイズ | 約 220 × 220 × 260 mm 以内 |
| 重量 | 2.5〜4.0 kg（片手で持ち運べる） |
| 材料 | PLA 主対象、PETG は v2（ヒートベッド追加）以降 |
| ノズル | 0.4 mm、最大 260 ℃ |
| 位置精度 | ±0.2 mm 程度 |
| 実用速度 | 25〜40 mm/s（最大 60 mm/s） |
| 操作 | スマホ / PC の Web UI から G-code 送信・監視（物理ディスプレイ不要） |
| 安全 | thermal runaway 検知、サーミスタ断線検知、緊急停止。携帯機ゆえ熱暴走リスクを軽視しない |

### ハード制約

- ファームウェアを自作しない。モーション制御・温調・安全機能は実戦検証済みの既存エコシステムに乗る
- 市販部品（制御基板、ホットエンド、エクストルーダ、リニアレール）を最大限流用し、独自設計はフレームとブラケット類に限定する
- フィラメントスプールは外付け（内蔵すると筐体が一気に大きくなる）

### このリポジトリでのスコープ

ForgeCAD でモデリングするのは機体の構造部分：フレームレイアウト（ブロックアウト）、
3D プリント製ブラケット・キャリッジ・ベッド支持・ケーブルガイド類、外装カバー。
電気設計・Klipper 設定は別途文書化するが CAD スコープ外。

## Approach

**Prusa Mini 式カンチレバー構成**。実績が最も厚く、80 mm 級では部品点数が最少になる。

- **Y 軸**: ベッドが前後に移動（bedslinger）
- **X 軸**: 片持ちガントリ上をプリントヘッドが左右に移動
- **Z 軸**: 片持ちガントリ自体が垂直コラムを昇降

```
        正面                          側面
┌─────────────────────┐      ┌──────────────────┐
│ ║ ← Zコラム(2020)    │      │  ║ Zコラム        │
│ ║══════════════     │      │  ║                │
│ ║   X軸ガントリ →    │      │  ╠══ X軸ガントリ   │
│ ║  [Head]           │      │  ║  [Head]        │
│ ║    ↓              │      │  ║    ↓           │
│ ║ ┌─────────┐       │      │ ┌──────────┐     │
│ ║ │Build Pl.│       │      │ │Build Pl. │←→ Y │
│ ║ └─────────┘       │      │ └──────────┘     │
│ ╚═╧═════════╧═══    │      │ ═╧══════════╧═   │
│   ベース(2020枠)     │      │  ベース           │
└─────────────────────┘      └──────────────────┘
```

制御は **Klipper アーキテクチャ**に乗る。Raspberry Pi Zero 2 W（ホスト）が軌道計算・
Web UI（Mainsail）・G-code 管理を担い、SKR Pico（RP2040 MCU）が時刻指定ステップパルス
実行・温調・エンドストップを担う。初版構想の「ESP32＝リアルタイム / Pi＝上位」という
責務分離の思想を、車輪の再発明なしに実現済みの形で採用したもの。

サブシステム構成：

| サブシステム | 採用 | 備考 |
|---|---|---|
| キネマティクス | Prusa Mini 式カンチレバー | 上記 |
| 制御基板 | SKR Pico（TMC2209 ×4 内蔵） | Klipper MCU |
| ホスト | Raspberry Pi Zero 2 W + Mainsail | v1 から搭載 |

| ホットエンド | 24V 小型 V6 互換、30〜40 W | |

| 電源 | 24V 6A (144 W) AC アダプタ | USB-C PD は v3 検討 |
| モータ | NEMA14 / NEMA17 パンケーキ ×4 | |
| リニアガイド | MGN9H（X/Y 150〜200 mm、Z 150 mm） | |
| Z 駆動 | T8 リードスクリュー ×1 + MGN9 | ガントリ昇降 |
| フレーム | 2020 アルミ + 3D プリント部品 | |

## Key Interfaces

| インターフェース | 内容 |
|---|---|
| Z コラム ⇔ ベースフレーム | 2020 押出材の直角結合。ガントリ荷重のモーメントを受ける最重要剛性点 |
| X ガントリ ⇔ Z キャリッジ | MGN9 キャリッジ 1〜2 個にプリント製ブラケットで片持ち固定。ヘッド全荷重＋モーメント |
| ヘッドキャリッジ ⇔ X レール | MGN9H キャリッジ。エクストルーダ＋ホットエンド＋ファン合計 ~250 g を想定 |
| ベッド ⇔ Y キャリッジ | 3 点支持（レベリングネジ）。輸送ロック機構をここに設ける |
| ホットエンド ⇔ エクストルーダ | 採用エクストルーダのマウント規格に従う（Orbiter 2 なら専用マウント） |
| 電源 ⇔ 本体 | DC ジャック 24V 入力。Pi 用に 24V→5V 降圧コンバータ |
| SKR Pico ⇔ Pi | UART 直結（USB ケーブル不要、配線がシンプル） |
| スプール ⇔ 本体 | 外付けホルダ。背面にフィラメント入口ガイド |

## Dictionary

| 用語 | 説明 |
|---|---|
| FDM | 熱溶解積層方式。フィラメントを溶かして層状に積む 3D プリント方式 |
| bedslinger | ベッド（造形台）自体が水平移動する構成の俗称 |
| カンチレバー構成 | X 軸ガントリを片側の柱だけで支える片持ち構造。Prusa Mini が代表例 |
| Klipper | ホスト（Linux）で軌道計算し MCU がパルス実行する分業型プリンタファームウェア |
| Mainsail | Klipper 用 Web UI。印刷操作・監視・設定をブラウザから行う |
| SKR Pico | BIGTREETECH 製の小型制御基板。RP2040 + TMC2209 ドライバ 4 軸内蔵 |
| TMC2209 | 静音ステッピングモータドライバ。センサレスホーミング対応 |
| MGN9 / MGN9H | 幅 9 mm のミニチュアリニアレール規格。H はキャリッジ長タイプ |
| T8 リードスクリュー | 直径 8 mm の台形ネジ。回転を直線運動に変換する Z 駆動の定番 |
| ダイレクトエクストルーダ | フィラメント送り機構をヘッドに直載せする方式。対義語は Bowden（チューブ経由で遠隔送り） |
| PEI プレート | ポリエーテルイミド表面の造形プレート。PLA は非加熱でも定着する |
| thermal runaway 検知 | ヒータ加熱中に温度が想定通り上がらない場合に異常（センサ脱落等）とみなし停止する安全機能 |
| input shaping | 機体の共振を打ち消すようモータ指令を補正する Klipper の振動抑制機能 |

## Alternatives

### キネマティクス

| 案 | 説明 | トレードオフ |
|---|---|---|
| A (採用) Prusa Mini 式 | ベッド Y、片持ち X、ガントリ Z 昇降 | 実績最多・部品最少・調整容易。片持ち剛性が弱点だが 80 mm 級・40 mm/s なら十分。bedslinger のベッド慣性も軽量ベッドなら無視できる |
| B ヘッド XY + ベッド Z 昇降（初版案） | Ultimaker 型の箱構成 | 剛性は出るがヘッドが 2 軸を持つガントリは独自設計要素が多く、小型筐体では調整性が悪い。Z ベッドを T8 ×1 + 片側ガイドで支えると首振り・傾きが出る |
| C CoreXY | ベルト 2 本の差動でヘッド XY | 高速・高剛性だがベルト経路が複雑で小型筐体では組立・調整コストが目的に見合わない |

### 制御アーキテクチャ

| 案 | 説明 | トレードオフ |
|---|---|---|
| A (採用) SKR Pico + Pi Zero 2 W + Klipper | ホスト/MCU 分業 | Web UI・メッシュレベリング・input shaping・PID チューン・安全機能が設定ファイルだけで揃う。Pi 分のコスト増は開発工数削減で回収 |
| B Marlin + STM32 系ボード（SKR Mini E3 等） | 単基板スタンドアロン | Pi 不要で構成最少。ただし Web UI・高度機能は追加ハード頼み。Pi を載せない要求が出た場合の次点 |
| C ESP32 独自ファーム（初版案） | ESP32-S3 で全制御を自作 | ステップ生成・look-ahead・温調 PID・thermal runaway を自作するのはプリンタ開発でなくファーム開発になる。Marlin の ESP32 ポートはメンテ薄、Klipper は upstream で ESP32 MCU 非対応。不採用 |

### エクストルーダ

| 案 | 説明 | トレードオフ |
|---|---|---|
| A (採用) 軽量ダイレクト | Orbiter 2 / Sherpa Mini 級（130〜200 g） | リトラクション調整が楽、PETG 安定、TPU も視野。実用 25〜40 mm/s・80 mm 角ではヘッド重量デメリットが顕在化しない |
| B Bowden（初版案） | チューブ経由の遠隔送り | ヘッドは最軽量になるがリトラクション調整がシビア。2020 年頃の小型機定石で、現在は軽量ダイレクトの完成度が上回る。PLA 専用最軽量派生としてなら残せる |

### 電源

| 案 | 説明 | トレードオフ |
|---|---|---|
| A (採用) 24V 6A (144 W) AC アダプタ | 標準 24V 系 | 一般プリンタ部品がそのまま使える。v2 ヒートベッド追加（合計 ~120 W）にも余裕 |
| B USB-C PD 20V 100W（初版案の携帯構成） | PD トリガで 20V 供給 | PD 100 W は上限であり連続常用は 80〜90 W が限度 → 非加熱ベッド専用。24V ヒータを 20V 駆動すると出力 69%（P∝V²）。携帯性は魅力なので v3 の派生仕様として再検討 |
| C バッテリー | 内蔵電池駆動 | 発熱体との相性が悪く重量増。不採用 |

## Usage Guide

### 組み立て（v1）

1. 2020 押出材でベースフレーム（約 200 × 200 mm 枠）を組む
2. Z コラム（2020、長さ ~240 mm）をベース後方寄りに垂直固定。コーナーブラケットで直角を出す ⚠️ ここの直角精度と剛性が造形品質を直接決める
3. Z コラムに MGN9 レールと T8 リードスクリューを取り付け、Z キャリッジ（プリント部品）を載せる
4. X ガントリ（2020 または アルミ板 + MGN9H レール）を Z キャリッジに片持ち固定
5. ベース上に Y 軸レール（MGN9H）とベルト駆動を敷き、ベッドキャリッジを載せる
6. ベッド（80〜100 mm PEI プレート）を 3 点レベリングネジでキャリッジに取り付け
7. ヘッドキャリッジにエクストルーダ＋ホットエンド＋ファンを組み付け、X レールに載せる
8. SKR Pico と Pi Zero 2 W をベース下部の電装ベイに固定、配線 ⚠️ ベッド Y 移動のケーブルとヘッド X 移動のケーブルに可動配線処理（ドラッグチェーンかスリーブ）が要る
9. 24V AC アダプタを接続、Pi に Klipper + Mainsail をセットアップ
10. printer.cfg を書き、PID チューン → Z オフセット調整 → ファーストプリント

### 印刷フロー

1. スマホ / PC のスライサ（OrcaSlicer 等）で G-code を作る
2. 同一 Wi-Fi 上の Mainsail にブラウザでアクセスし、G-code をアップロード → 印刷開始
3. 温度・進捗は Mainsail で監視。異常時は Web UI か本体の緊急停止ボタンで停止
4. 印刷完了後、PEI プレートを撓ませて部品を剥がす

### 持ち運び

1. 印刷していないこと・ノズルが冷えていることを確認
2. ベッドの輸送ロックを掛ける（Y キャリッジを固定）⚠️ ロック機構は要設計。忘れるとベルト・キャリッジに衝撃が入る
3. AC アダプタとスプールを外し、本体（~3 kg）を片手で運ぶ ⚠️ 持ち手（グリップ）を筐体設計に含めるか要検討

## Concerns

1. **片持ちガントリの剛性**: ヘッド ~250 g（軽量ダイレクト採用時）が X 端にいるとき、Z キャリッジ結合部のモーメントで先端が撓む。Prusa Mini 実績から 80 mm 級では成立見込みだが、プリント製ブラケットの肉厚・リブ設計が要点。目標：ヘッド自重による先端変位 0.1 mm 以下
2. **Z コラム直角度**: 組立時の直角出しがユーザー任せになる。ブラケットに突き当て基準面を設けて再現性を確保したい
3. **可動配線**: ヘッド（X）・ベッド（Y）・ガントリ（Z）の 3 系統の可動ケーブル処理。小型筐体では曲げ半径が取りにくい
4. **重量予算**: 2020 フレーム＋電装＋モータ 4 個で 2.5〜4.0 kg に収まるかは未検証。フレーム部材長を確定した時点で質量予算を forge-verify の `suite.budget()` でチェックする
5. **本体 220 × 220 × 260 mm への収まり**: bedslinger は Y 移動分の前後スペースが要る（ベッド 100 + ストローク 80 + マージン）。ベース 200 mm 角で足りるか、レイアウトのブロックアウトで先に検証する
6. **輸送ロック機構**: 未設計。Y キャリッジ固定に加え、Z ガントリの自重落下防止（T8 のセルフロックで足りるか）も確認する
7. **非加熱 PEI での PLA 定着**: 室温が低い出張先環境で定着不良の可能性。ブリム標準運用か、v2 前倒しでの低温ベッド追加かは v1 の実測で判断
8. **Pi Zero 2 W の供給性と性能**: input shaping の共振測定（ADXL345）実行時にメモリが厳しい場合がある。代替は Pi 3A+ / CB1
9. **Klipper の ESP32 非対応の一次確認**: レビューで「upstream 非対応」と判断したが、Klipper リポジトリの `src/Kconfig` での一次確認は未実施（採用案では ESP32 を使わないため優先度低）

## Decisions

2026-07-02 のクロスレビュー（Claude + Codex、双方の指摘が一致）で初版構想から変更した決定。

| # | 決定 | 根拠 |
|---|---|---|
| 1 | ファームは自作せず Klipper（SKR Pico + Pi Zero 2 W）を採用。ESP32 独自ファームは不採用 | 代替案・制御 C 参照。安全機能を自作するリスクと開発工数が本体開発を圧迫する。初版の責務分離思想は Klipper のホスト/MCU 分業がそのまま実現している |
| 2 | キネマティクスは Prusa Mini 式カンチレバーに確定 | 代替案・キネマティクス B/C 参照。実績・部品点数・調整容易性で優位。初版の Z 昇降ベッド案は片側支持の首振りリスクと独自設計要素が多い |
| 3 | エクストルーダは軽量ダイレクトを標準に。Bowden は派生案に格下げ | 代替案・エクストルーダ B 参照。目標速度域ではヘッド重量デメリットが出ず、調整容易性・材料対応で勝る |
| 4 | 電源は 24V 6A AC アダプタ。USB-C PD は v3 の派生仕様に先送り | 代替案・電源 B 参照。PD 100 W は連続 80〜90 W が限度で v2 ヒートベッドと両立しない |

以下は今後のレビューで決める。

| # | 未決事項 | 対応 |
|---|---|---|
| 5 | フレームレイアウトの成立性（Concern 4, 5） | ブロックアウトモデルを作って寸法・質量を検証してから確定 |
| 6 | エクストルーダの具体機種（Orbiter 2 か Sherpa Mini か） | 入手性と価格で決定。マウント設計に影響 |
| 7 | 輸送ロック・持ち手の要否と方式（Concern 6、Usage Guide ⚠️） | ブロックアウト後に検討 |
