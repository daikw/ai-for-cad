# mini-humanoid v2 パーツ台帳（並列作業の契約書）

v1 のブロックアウトを ref 画像の造形密度へ引き上げる。参照: `refs/ref-1-ortho-front-side.png`（プロポーション正）、`refs/ref-3-appearance.png`（外装意匠正）、`refs/ref-2-servo-layout.png`（内部）。

## 凍結事項（全エージェント共通・変更禁止）

- `dims.js` と `servo-xl330.forge.js` は **凍結**。関節 Z スタック・サーボ配置・モジュール間インターフェースは v1 のまま
- 各モジュールの返り値契約は不変:
  `{ group, solids: () => ({ printed: [...], servos: [...] }) }`
  （torso は `fixed / frameOnly / pelvisOnly / board / battery` キーも維持）
- **全高 = 300.0mm 厳守**（頭頂 Z=300、足裏 Z=0）。checks が 300±3 でゲートする
- ブーリアンは kernel-pitfalls 準拠: difference は必ずプリミティブ基底に対して行う。union 派生基底への difference、同一プリミティブへの difference+intersection 連鎖は禁止（`playgrounds/forgecad/lib/forge-verify/kernel-pitfalls.md`）
- グループ間の貫通は禁止（checks が pairwise ≤1mm³ でゲート）。パーツ間の意図的溶着（ボルト結合表現）は同一グループ内のみ、1 箇所 ≤500mm³ 目安でコメントに明記

## 空間予算（グループ間クリアランスの担保）

| グループ | ローカル占有許容域 |
|---|---|
| Leg（±26 配置） | x∈[-23,23], y∈[-36,42], z∈[0,142]。hip roll 本体のみ骨盤ポケット内 z≤160 可 |
| Arm | z∈[110,240]。z<140 の部分（手首・ハンド）は \|x\|≥42.5。x<36 に入るのは肩ピッチサーボと肩ブラケットの貫通部のみ |
| Torso | 外形 x∈[-36,36], y∈[-23,23], z∈[142,252]（シェル追い込みで ±2mm 拡張可） |
| Head | z∈[250,300], x∈[-38,38], y∈[-35,35]。首サーボは v1 位置固定 |

全体エンベロープ: x∈[-85,85], y∈[-50,70], z∈[0,305]（checks が外殻法でゲート）

## 質量予算（印刷部品、実効密度 0.8 g/cm³ で体積計上）

head ≤45g / torso ≤110g / leg ≤55g×2 / arm ≤25g×2 → 印刷計 ≤315g、総質量 ≤800g。
シェルは肉厚 2.0–2.5mm の殻にして体積を稼がない（ベタ詰め禁止）。

## スタイル規範

- ツートーン: 白シェル `#e6e2da`（matte, roughness 0.6）/ メカ部 `#23262b`・`#4b5158`
- シェル類はフィレット/面取り必須（角丸が v1 との最大の差）。fillet は対象エッジを絞る（全エッジ fillet の乱発禁止、OCCT/render 時間に注意）
- 教材ラベル・矢印・文字は入れない。実機に存在する意匠（パネルライン、ネジ頭など）のみ可

## パーツ仕様（担当別）

### A. 頭部（head.forge.js 所有）

ref-3 が正。v1 の楕円体を廃し:

1. **ヘルメット殻**: 角丸ボックス系（W72×D66×H50 目安、大 R のフィレットまたはロフト）。頭頂 Z=300 厳守
2. **フェイスプレート**: 黒、面一〜わずかに凹、全幅の角丸窓（ヘルメット前面の 6 割幅）
3. **目 ×2**: 白発光の丸目（ref-3）
4. **耳ポッド ×2**: 側面の暗色円盤（Ø30 前後、薄い凸）
5. **首**: v1 の neck servo / 首筒 / スカル neck passage 相当の機構は維持

### B. 脚（leg.forge.js 所有）— 最重量ワークストリーム

ref-1 側面が正。v1 の平板を置換:

1. **足**: 2 層造形。白アッパー（爪先へ向けて絞るくさび形＋面取り）+ 黒ソール薄板。踵の小パーツ可。外形は現行 45×65 から ±3mm 程度まで
2. **足首 U リンク（白）**: ref-1 で目立つコの字リンク。足首ピッチホーンを両持ちで挟む造形
3. **膝 U リンク（白）**: 同様に膝ホーンを挟む。v1 の側板 4 枚はこれに置換または統合
4. **大腿シェル / すねシェル（白）**: サーボを前面から包むラップ形状（貼り板ではなく殻）
5. サーボ 5 個・関節 Z・ホーン位置は v1 のまま

### C. 胴体（torso.forge.js 所有）

ref-1 正面が正:

1. **胸プレート**: 台形パネルライン入り（ref-1 の上胸/腹の 2 分割意匠）、面取り
2. **背中シェル**: side ビューのバックパック状の膨らみ
3. **骨盤カバー**: 黒ブロックの前面に白 or 濃灰の意匠面
4. キャビティ・電装配置・肩穴/首穴は v1 のまま（フレーム機能は維持したままシェル意匠を彫る）

### D. 腕（arm.forge.js 所有）

ref-3 が正:

1. **肩ポッド（白）**: 肩上面を覆う丸みのあるカバー
2. **上腕シェル / 前腕シェル（白）**: 面取り付きラップ殻。前腕は先細り
3. **ハンド**: C 爪を維持しつつ厚み・面取り・指先の丸みを追加（ref-3 の暗色グリッパ）
4. サーボ 3 個・関節 Z は v1 のまま。z<140 は \|x\|≥42.5 厳守（大腿クリアランス）

### E. 検証・シーン（checks.forge.js / main.forge.js 所有）

1. checks.forge.js: 既存チェックを維持したまま、(a) 全印刷パーツの bed-fit ループ（solids().printed を走査）、(b) 質量予算の部位別内訳表示、(c) 頭頂 Z=300±1 の追加チェック
2. main.forge.js: シーン調光の仕上げ + ref 一致ビュー追加（`refFront`: ref-1 正面相当の望遠正面、`refHero`: ref-3 相当の 3/4 俯瞰）。モジュール合成・グループ名は変更禁止
3. 数値の最終確定は統合後に再実行（v1 ジオメトリでのスクリプト動作確認まで）
