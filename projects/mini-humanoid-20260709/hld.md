## Problem

デスクトップサイズ（全高 300mm ± 10mm）の二足歩行ヒューマノイドロボットを設計する。

ハード要件:

1. **全高 300mm ± 10mm**（直立ニュートラル姿勢、足裏〜頭頂）
2. **二足歩行可能な自由度構成**であること（静歩行ベース。ROBOTIS Mini 級の運動性）
3. **市販サーボモータを使用**し、選定根拠（トルク予算）を示すこと
4. **フレームは個人が製作可能**であること（ワンオフ前提）
5. 制御基板・バッテリーを**胴体内に収納**できること
6. 転倒に耐える構造（頭部・胸部シェルで基板を保護）

やわらかい要件（参考画像から）:

- ref-1/ref-3 のプロポーション: やや大きめの頭、コンパクトな胴、足裏は大きめのフラットプレート
- ref-3 の外観: 白基調のシェル + ダークグレーのメカ部、丸みのある頭部に黒いフェイスパネル

## Approach

**「サーボブラケット直列連結」構成**。ホビーヒューマノイドの定石で、サーボ本体を構造部材として使い、U 字/L 字ブラケットで直列に連結して四肢を構成する。外装シェルは構造から独立した化粧パネルとして被せる。

自由度（DoF）構成は **17 DoF**:

```
                 [頭部 Ø72mm]
                    │ ← 首ヨー ×1
              ┌─────┴─────┐
   肩ピッチ → ●  胴体      ● ← 肩ピッチ
   肩ロール → ●  90H×72W  ● ← 肩ロール
              │  ×46D     │
   肘     →  ●           ● ← 肘          腕: 3 DoF ×2 = 6
              │           │
             (手)  ┌───┐  (手)
              ┌────┤腰 ├────┐
   股ロール → ●    └───┘    ● ← 股ロール
   股ピッチ → ●             ● ← 股ピッチ
              │ 大腿 52     │
   膝     →  ●             ● ← 膝
              │ 下腿 52     │
   足首ピッチ→ ●             ● ← 足首ピッチ  脚: 5 DoF ×2 = 10
   足首ロール→ ●             ● ← 足首ロール
           ┌──┴──┐       ┌──┴──┐
           │足 65L│       │足 65L│         首: 1 DoF
           └─────┘       └─────┘          計 17 DoF
```

垂直スタック（目標: 合計 300mm）:

```
頭部         62   ← 首サーボ含む
胴体         92   ← 制御基板 + バッテリー内蔵
骨盤ブロック  24
大腿         52
下腿         52
足首・足     18
            ───
            300 mm
```

- 股ヨーは省略（ROBOTIS Mini と同じ。旋回は足踏みシャッフルで行う）
- 手は固定爪（非駆動）。ref-1 の開いた爪形状を模す

### サーボ選定（トルク予算）

**質量予算**: 総質量 ≤ 800g

| 項目 | 概算 |
|---|---|
| サーボ 17 個 × 18g | 306g |
| フレーム・ブラケット（PETG 印刷） | ~200g |
| バッテリー 2S LiPo 800mAh | ~45g |
| 制御基板 + 配線 + ファスナー | ~80g |
| 外装シェル | ~70g |
| **合計** | **~700g** |

**最悪ケース・トルク見積り**（総質量 800g で安全側に計算）:

| 関節 | シナリオ | 支持質量 | モーメントアーム | 必要トルク | 必要マージン×2 |
|---|---|---|---|---|---|
| 膝 | しゃがみ姿勢から立ち上がり | ~0.60kg | ~45mm | 0.26 N·m | 0.53 N·m |
| 足首ピッチ | 片脚支持で CoM 前後補正 | ~0.75kg | ~30mm | 0.22 N·m | 0.44 N·m |
| 股ピッチ | 前傾からの復帰 | ~0.45kg | ~40mm | 0.18 N·m | 0.35 N·m |
| 肩ピッチ | 腕水平持ち上げ | ~0.08kg | ~85mm | 0.07 N·m | 0.13 N·m |

→ **脚関節に 0.5 N·m 級のストールトルクが必要**。マージン 2× はストール連続運転を避けるための実用下限。

**選定: ROBOTIS Dynamixel XL330-M288-T ×17**

| 項目 | XL330-M288-T |
|---|---|
| ストールトルク | 0.52 N·m @ 5.0V |
| 質量 | 18g |
| 寸法 | 20 × 34 × 26.5 mm |
| 電源 | 3.7–6.0 V |
| 通信 | TTL 半二重デイジーチェーン（Protocol 2.0） |
| フィードバック | 位置・速度・電流・温度 |
| 参考価格 | ~$24 → 17 個で ~$410 |

膝・足首のマージンは 2.0× 弱とタイトだが、ROBOTIS Mini（XL-320、0.39 N·m、660g 級）の実績よりも余裕がある構成。全関節同一型番にすることで補修部品・制御パラメータを単純化する。

### 電装

- **制御基板**: ROBOTIS OpenRB-150（XL330 専用設計、TTL バス直結、Arduino 互換、47×39mm）
- **電源**: 2S LiPo 800mAh → 5V/5A UBEC 経由でバス給電（XL330 は 6V 上限のため直結不可）
- **配線**: デイジーチェーンで四肢ごとに 1 系統（腕 L/R、脚 L/R、首）→ 胴体内ハブへ

## Key Interfaces

| インターフェース | 内容 |
|---|---|
| サーボホーン ↔ ブラケット | XL330 標準ホーン穴パターン（M2）。全関節共通 |
| サーボ本体 ↔ フレーム | XL330 側面 M2 タップ穴 4 点。ブラケット側は貫通穴 |
| 胴体 ↔ 骨盤 | 骨盤ブロック上面に M2.5 ×4。胴体下面プレートと締結 |
| 胴体内部 ↔ 基板/バッテリー | OpenRB-150 (47×39mm) を背面側、バッテリー(50×30×15mm)を腹側に平積み |
| 足裏 ↔ 地面 | 65 × 45mm フラットプレート。静歩行の支持多角形を確保 |
| シェル ↔ フレーム | スナップフィット爪 + M2 ネジ併用（頭・胸・すね） |

## Dictionary

| 用語 | 説明 |
|---|---|
| DoF | Degree of Freedom（自由度）。駆動関節 1 つ = 1 DoF |
| ストールトルク | サーボが停止状態で出せる最大トルク。連続運転はこの 1/2 以下が目安 |
| デイジーチェーン | サーボを数珠つなぎに配線し 1 本のバスで制御する方式 |
| U 字ブラケット | サーボ出力軸と反対側アイドラを両持ちで挟むコの字金具。関節の標準構成 |
| 静歩行 | 常に重心投影が支持多角形内にある歩き方。ZMP 制御不要で低速 |
| 支持多角形 | 接地している足裏が張る凸包。この中に重心があれば倒れない |
| UBEC | スイッチング式降圧電源。LiPo 電圧をサーボ用 5V に変換する |
| ホーン | サーボ出力軸に付く円盤/アーム。ここに部材をネジ止めする |

## Alternatives

### サーボ

| 選択肢 | 説明 | トレードオフ |
|---|---|---|
| **A. Dynamixel XL330-M288（推奨）** | 0.52 N·m/18g のスマートサーボ。TTL バス、フルフィードバック | 高価（~$410/17個）だが配線激減・電流監視で保護可能。この級の研究/ホビー機のデファクト |
| B. FEETECH STS3032 | 0.4 N·m/20g のバス式サーボ。~$8/個 | 安いがフィードバック精度・ファーム品質が落ちる。総額 ~$140 で予算最優先ならあり |
| C. MG90S + PCA9685 | PWM マイクロサーボ 13.4g、~$3/個 | 最安だが 17ch の PWM 配線が地獄、位置フィードバック無し、ストール検知不可で焼損リスク |
| D. Dynamixel XL-320 | ROBOTIS Mini 実績品 | 実質ディスコン。トルクも XL330 に劣る。新規採用の理由なし |

### フレーム製法

| 選択肢 | 説明 | トレードオフ |
|---|---|---|
| **A. FDM 印刷 PETG（推奨）** | ブラケット・シェルとも印刷。壁厚 2.4mm 以上 | ワンオフに最適。複雑なシェル形状が自由。剛性は膝・股ブラケットのリブで補う |
| B. アルミ板金ブラケット + 印刷シェル | 構造部は 1.5mm A5052 曲げ | 剛性最良だがワンオフの曲げ外注はリードタイム・コスト増。30cm 級では過剛性 |
| C. 全 CNC 削り出し | POM/アルミ | 精度最良・最高価。ホビーワンオフでは非現実的 |

### 頭部センサ

| 選択肢 | 説明 | トレードオフ |
|---|---|---|
| **A. ダミーカメラ窓のみ（推奨）** | 本モデルでは外形のみ。後日カメラ搭載可能な開口を確保 | スコープ制御。CAD としては窓形状だけ表現 |
| B. ESP32-CAM 内蔵 | 頭部に実カメラ | 配線・発熱・首サーボ負荷が増える。v2 で検討 |

## Usage Guide（組立の流れ）

1. XL330 ×17 に ID を書き込む（脚 L: 1–5、脚 R: 6–10、腕 L: 11–13、腕 R: 14–16、首: 17）
2. 足プレートに足首ロールサーボを固定し、足首ピッチサーボと U 字ブラケットで直交連結する
3. 下腿フレーム（印刷）で足首ピッチ ↔ 膝サーボを接続する
4. 大腿フレームで膝 ↔ 股ピッチを接続し、股ロールサーボを骨盤ブロックへ固定する
5. 骨盤ブロックを胴体下面プレートに M2.5 で締結する
6. 腕（肩ピッチ → 肩ロール → 肘 → 固定爪ハンド）を組み、胴体側面に肩ピッチサーボを固定する
7. 胴体フレームに OpenRB-150（背面）と LiPo（腹側）を搭載し、四肢のバス配線をハブへ集約する ⚠️ 配線通路を胴体フレームに設けないと組立時に挟み込む
8. 首サーボを胴体上面に固定し、頭部シェルを被せる
9. 胸・すね・頭のシェルパネルをスナップフィットで装着する ⚠️ シェルとブラケット可動域の干渉チェックが必要（特に肩ロール）
10. USB で OpenRB-150 に接続し、ニュートラル姿勢キャリブレーション → 歩行モーション転送

## Concerns

1. **膝・足首のトルクマージンが 2.0× ぎりぎり**。しゃがみ込みの深い姿勢（膝 90° 超）を制御側で禁止しないとストール域に入る。モーション設計時の関節リミットで担保する
2. **足首ロール軸が地面から 18mm と低く**、U 字ブラケットの足裏側クリアランスがタイト。足首ロール可動域は ±25° に制限される見込み
3. **肩ロールサーボと胸シェルの干渉**。腕を真横に上げる際、シェル開口を大きく取るか可動域を ±80° に制限する必要がある（⚠️ from Usage Guide 9）
4. **胴体内の配線容積**。バス配線 4 系統 + 電源で、基板・バッテリーの隙間 ~8mm に収める必要がある（⚠️ from Usage Guide 7）
5. **XL330 のホーン規格は 2.5mm ピッチの M2**。印刷ブラケットの穴精度（FDM で ±0.2mm）で複数穴の同時嵌合が渋くなる可能性 → 穴径 +0.2mm 補正を LLD で規定する
6. **総質量 700g 見積りの妥当性**。フレーム 200g / シェル 70g は印刷条件（インフィル 30%）依存。CAD 側で体積 × 密度の質量予算チェックを入れて検証する
7. **静歩行の成立性**は CAD では足裏面積と CoM 位置でしか確認できない。片脚支持時に CoM が足裏 65×45mm に入ることを重心計算でチェックする

## Decisions

| # | Decision | Rationale |
|---|----------|-----------|
| 1 | 17 DoF（脚 5×2 + 腕 3×2 + 首 1）、股ヨーなし | ROBOTIS Mini 実績構成。サーボ数と歩行能力のバランス。Concern 対象外の旋回はシャッフルで代替 |
| 2 | サーボは XL330-M288-T ×17 で統一 | トルク予算 0.5 N·m 級を満たす唯一の軽量スマートサーボ。Alternatives A 採用 |
| 3 | フレーム・シェルとも FDM 印刷 PETG | ワンオフ・複雑形状・十分な剛性。Alternatives A 採用 |
| 4 | 制御は OpenRB-150 + 2S LiPo + 5V UBEC | XL330 バス直結の純正組合せ。6V 上限のため UBEC 必須 |
| 5 | 頭部はダミーカメラ窓のみ（v1） | スコープ制御。Alternatives A 採用 |
| 6 | 肩ロール可動域 ±80°、足首ロール ±25°、膝 ≤90° を設計前提とする | Concern 1–3 をジオメトリ側で吸収せず運用リミットで担保 |
| 7 | 質量予算 800g・CoM チェックを checks.forge.js の budget に含める | Concern 6, 7 を数値検証でゲートする |
