## 2. 公開された実プロンプト（ワンショット設計の到達点）

7/9 の投稿で、佐々木氏が Fable5 に渡した**実際の指示プロンプト全文**が公開された。これが「1時間待ちワンショットで回路+レイアウトまで到達する」入力の粒度:

> XIAO ESP32 S3をベースに 液晶には、HS20HS072RX ( C5329582) を搭載したハンディTerminalを開発, Displayは横向きに配置 QweryKey配置と、MEMS Mic (C2840615 : 自分の在庫使います), Speaker(端子はC10819)も装着 XIAO-ESP32-S3には充電回路も入っているので　LiPoはXIAO-ESP32-S3と繋ぎます。

構造を分解すると:

1. **ベースモジュールの指名**（XIAO ESP32-S3）— MCU 周辺回路の設計を丸ごと省略
2. **主要部品を LCSC C 番号で指定**（C5329582 / C2840615 / C10819）— 部品選定の曖昧さを消し、JLCPCB PCBA 発注可能性を入力時点で担保
3. **物理配置の制約を一言**（Display 横向き、QWERTY 配置）
4. **在庫方針**（「自分の在庫使います」= PCBA 実装対象から除外）
5. **既存回路の流用指示**（充電回路は XIAO 内蔵 → LiPo 直結）

**注目点**: 高度な回路知識ではなく「C番号 + モジュール流用 + 配置一言」という、素人でも書ける粒度に落ちている。これを支えているのが「EasyEDA Skill + ノウハウを書き込んだ独自 Skill」。

## 3. 確立されたパイプライン（画像から復元）

```
[ネタ出し]   Fable5 が次の基板ネタをスコアカード（新規性/×9の意味/コスト/既存アーキ流用）で提案
     ↓        └ JLCPCB 在庫を確認しながら候補を出す ("Checking deep-MEMS candidates stock")
[回路設計]   Claude Code + EasyEDA Pro skill（〜5分/シールド）
     ↓
[レイアウト]  同上（〜3分/シールド、2層。デジタル回路なら Auto Routing 逃げも可）
     ↓
[発注]       JLCPCB PCB+PCBA（HandyTerminal 実例: PCB 5枚 $2 + PCBA 2枚 $128.86 = $148.88、アイディア→発注 1.5h）
     ↓
[FW 開発]    Fable5 が「回路図を読んで」ESP-IDF (mcp) でファームウェア生成
     ↓
[アプリ]     XCode mcp で BLE 連携の管理用 iOS アプリまで生成
     ↓
[検証]       実基板到着 → 動作確認 → 不具合の切り分け（AI 起因 / 人間の指示ミス）→ 再設計ループ
```

- 生産性の実測: **基板総数 20 枚超 / 1日 2-3 枚の新規設計**（7/9 時点）
- シリーズ化: 「Matrix Five ファミリー」— ESP32-S3 DevBoard + 8種のセンサーシールド（各センサー ×9 のマトリクス構成）。#2 IMU×9、#3 BME690、#4 MLX90393×9（= Meta ReSkin 相当の磁気触覚）、#7 近接センサー、#9 MagPix Coil 3×3（PCB 銅箔スパイラルコイル = 部品コストほぼゼロのセンサー）
- 出口: Humanoid の指の触覚センサー評価プラットフォーム（HumanoidFive アームと接続予定）

## 4. 深い洞察 — スキル化に効く教訓

### 4.1 「AI ができる領域」の境界線が明確に語られている

| 領域 | AI (現状) | 根拠ツイート |
|---|---|---|
| 回路設計（デジタル・ブレークアウト級） | ◎ ワンショット可 | 7/3, 7/9 |
| 回路設計（電源・モータードライバー） | △ 配線・配置が厳しい | 7/3 20:35 |
| PCB レイアウト（2層・低速信号） | ○ 3分で生成、ただし品質は「ネットリストだけの配線」と自嘲 | 7/3, 7/9 11:56 |
| アートワーク（高密度・高速・EMC） | ✗ **特化 AI が必要**（スタートアップ待ち） | 7/10 01:01 |
| 回路図からの FW 生成 | ◎ 「本当に新しいユーザ体験」 | 7/9 12:00 |
| 部品選定（在庫連動） | ◎ JLCPCB 在庫から候補提示→即発注 | 7/3, 7/6 |
| 物理的な検証 | ✗ 人間必須（FFC コネクタの向きミスは実物が来るまで気づけない） | 7/9 23:46 |
| 製造業者とのやり取り | ✗ 暗黙知の塊 | 7/10 00:48 |

→ スキル化の主戦場は「◎と✗の間にある△」。**具体的には (a) 発注前 DRC 的チェックリスト（コネクタ極性・FFC 向き・ピン 1 マーク）、(b) 電源系の設計パターン集、(c) 業者とのやり取りの定型知識**。

### 4.2 検証ループの非対称性が最大のボトルネック

ソフトウェアと違い**「コンパイル→実行」に4〜7日と $150 かかる**。今週の失敗例は2つとも「実物が届くまで検出不可能」だった:

- Opus4.8 基板: Display は動いたがボタン認識せず（回路 or FW の切り分けが実物依存）
- Fable5 基板: 人間の指示ミスで FFC コネクタの向きが逆

これは ForgeCAD で「checks.forge.js が ALL PASS するまで発注しない」という本リポジトリの検証ゲート思想と正確に同型。**PCB 版の forge-verify（発注前に機械検証できる項目を最大化するチェックスイート）が、スキル化の最も価値ある部分**。氏も「100基板作らないと良し悪しわからない」= 経験則がまだ言語化されていないと認めている。

### 4.3 スキルこそが現在の競争力（本人が明言）

- 「PCB系は専門用語とノウハウの塊だから、言語化が難しい。**ここら辺をskill化できればワンショットでいける**」(7/10)
- 「今は skill で補充していくしかない。操作データ学習は先に中華 LLM がやるだろう」(7/10)
- 実際に「EasyEDA Skill + **諸々のノウハウを書き込んだ独自 Skill**」の2階建てで運用している (7/9)

→ 氏の構成は本リポジトリと同じ「汎用 EDA 操作スキル + ドメインノウハウスキル」の分離。われわれの easyeda-api / easyeda-pro-claude-skill / jlcpcb-parts-search は前者と部品調達をカバー済み。**欠けているのは後者 = 設計ノウハウ・発注前チェック・プロンプト様式のスキル**。

### 4.4 モデル比較の実地評価（EDA 操作という「未知ツール」ベンチマーク）

- **Fable5**: 回路設計で軍配。「未知なるツール EasyEDA を着々と使いこなす」「頼んだものを文句なく作り続ける」
- **Opus 4.8 / 4.6**: 実基板まで到達実績あり（Display 動作 OK、ボタン NG）
- **Codex 5.6 Sol**: 進行は丁寧になったが「右往左往して仕上がらない」、5.5 より悪化と評価
- 重要な相対化: 「**まずは頼む人間側が最適化していかないといけない**」— モデル差よりプロンプト+スキルの整備が支配的、という結論

### 4.5 経済性と地政学

- 1 サイクル $150・1.5 時間で「アイディア→発注」。トークン費用を含めても個人が1日2〜3枚回せる
- 「格安 Token もハードウェア作りもいつストップされるかわからない。**作れるうちに作れるだけ作る**」— JLCPCB/LCSC という中国サプライチェーンと安価な推論への二重依存を自覚した上でのスピード優先戦略
- JLCPCB 側に「AI 生成ネットリスト配線」が殺到し混雑している(らしい)という観測 — 製造側の受け入れ工程が新たなボトルネックになる兆候

## 5. 本リポジトリへの示唆 — スキル化バックログ（提案）

1. **pcb-preorder-checks**（最優先）: 発注前の機械検証チェックリスト。コネクタ極性・FFC の向きと接点面・ピン 1 整合・シルク・電源レール導通・アンチパターン検出。forge-verify の「数値ゲートを通らないと完了ではない」思想を EasyEDA ネットリスト（.tel エクスポート + grep/diff）に適用する
2. **pcb-prompt-brief**: §2 のプロンプト様式をテンプレート化 —「ベースモジュール指名 + LCSC C番号 + 配置制約 + 在庫方針 + 流用指示」の 5 要素ブリーフ。forgecad-prepare-prompt の PCB 版
3. **pcb-design-knowhow**: 電源回路・モータードライバー等「AI が苦手」と名指しされた領域のパターン集。Auto Routing に逃げる判断基準を含む
4. **回路図→FW ブリッジ**: EasyEDA 回路図データ → ESP-IDF ファームウェア生成の接続（氏は ESP-IDF mcp + XCode mcp で実証済み。うちのスタックでは easyeda-export-artifacts の拡張ポイント）
5. **jlcpcb-parts-search との接続**: 「在庫があるセンサーから候補を出す→そのまま発注」の在庫連動ネタ出しは、既存の jlcpcb-parts-search スキルにスコアカード形式（新規性/コスト/設計資産流用）のネタ出しモードを足すと再現できる
