# 次基板ネタ出しスコアカード

「次に何の基板を作るか」を、思いつきの列挙でなく 4 軸のスコアカードで提案する
ためのフォーマット。提案する部品は**必ず `jlc-search.sh` で在庫を裏取り**して
から載せる（在庫のない部品を軸にしたネタは実体化できない）。

このパターンは @gclue_akira の実運用（Claude Code が JLCPCB 在庫と連動して
次基板を提案 → そのまま発注）で実証されたもの
（<https://x.com/gclue_akira/status/2073013336187535561> ほか、2026-07）。

## 4 軸

| 軸 | 見るもの | 目安 |
|---|---|---|
| **新規性** | 同じ構成が既に世の中/手元にあるか。「場・分布を測る」「銅箔そのものをセンサーにする」等の切り口 | ◎ 誰もやっていない / ○ 変種 / △ 定番 |
| **部品コスト** | 主要部品の単価 × 個数（PRICE@1 で概算）。銅箔パターン流用なら「ほぼ基板代のみ」 | ◎ <$5 / ○ <$20 / △ それ以上 |
| **設計資産の流用** | 既存基板の回路ブロック・バス構成・フットプリントをどれだけコピーできるか | ◎ そのまま / ○ バス流用 / △ 新規 |
| **在庫** | LCSC 在庫数と Basic/Extended 区分（`search --class basic` / `part CXXXX` で確認） | ◎ Basic 潤沢 / ○ Extended 潤沢 / △ 薄い |

## 提案の書式

3〜5 案を表にし、各案に「採用したら最初に確定する C 番号」を添える:

```
| 案 | 新規性 | 部品コスト | 設計資産流用 | 在庫 | 主要部品 |
|---|---|---|---|---|---|
| 気圧マトリクス | ◎ 圧力の"場" | ◎ 安い | ◎ SPI+CS×N そのまま | ◎ C部番(Basic, 在庫n) | BMP581 ×9 |
| ...
```

推し 1 案を選んで理由を 2〜3 文で述べ、採用されたら pcb-design-brief に
引き継ぐ（スコアカードの主要部品欄がそのままブリーフの②になる）。

## コツ

- 制約付きの依頼（「MEMS 縛り」「触覚センサー系」）はスコアが上がる軸を
  明示してから探すと発散しない
- 「×N マトリクス化」は新規性を稼ぎやすい定石（点の測定 → 場・勾配の測定に
  変わる）。ただし MCU 側のペリフェラル上限（I2C アドレス・CS 本数・PDM ch）を
  先に確認する
- I2C 部品はアドレス可変性（ピン設定 / XSHUT）を在庫確認と同時に見る。
  固定アドレス ×N は電源スイッチ等の回避策が必要になりコスト軸を悪化させる
