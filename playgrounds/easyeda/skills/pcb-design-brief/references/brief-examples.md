## 実例: ハンディターミナル

> XIAO ESP32 S3をベースに 液晶には、HS20HS072RX ( C5329582) を搭載した
> ハンディTerminalを開発, Displayは横向きに配置 QweryKey配置と、
> MEMS Mic (C2840615 : 自分の在庫使います), Speaker(端子はC10819)も装着
> XIAO-ESP32-S3には充電回路も入っているので　LiPoはXIAO-ESP32-S3と繋ぎます。

### 5要素への分解

| 要素 | 該当箇所 | 省略された設計作業 |
|---|---|---|
| ①ベースモジュール | 「XIAO ESP32 S3をベースに」 | MCU 周辺・USB・書き込み・電源回路の全設計 |
| ②LCSC C番号 | C5329582 / C2840615 / C10819 | 部品選定・フットプリント探索・在庫確認の往復 |
| ③配置制約 | 「Displayは横向きに配置」「QweryKey配置」 | レイアウト方針の推測・やり直し |
| ④在庫方針 | 「自分の在庫使います」 | PCBA 実装対象の判断（DNP 指定漏れ） |
| ⑤流用指示 | 「充電回路も入っているので LiPoは…と繋ぎます」 | 充電 IC の重複設計 |

### この実例に足りなかったもの（教訓）

同氏は後日 FFC コネクタの向きミスを実物到着後に発見している

③配置制約に「FFC の接点面・挿入方向」が含まれていなかったことが原因の一つ。
テンプレートではこれを必須欄に昇格させている。

## 粒度の目安

- **十分**: モジュール名 + C 番号 + 配置一言。回路知識のない発注者でも書ける
- **不足**: 「ESP32 で液晶付きの端末」— 部品が未確定で、設計パスが選定から
  始めることになり、在庫外部品・フットプリント不明部品を掴むリスクが上がる
- **過剰**: ピン割当表・配線指示 — 設計パスの裁量を奪い、モジュール実装との
  矛盾を作りやすい。ピン割当は回路図確定後に FW 側が読む方が安全
