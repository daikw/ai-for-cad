# 発注前チェックリスト

Adapted from [Keitark/pcba-design-skills](https://github.com/Keitark/pcba-design-skills)
(MIT) `pcb-layout-review` / `release-pcba-fabrication` の各チェックリスト —
EasyEDA Pro + JLCPCB の 2〜4 層プロトタイプ向けに翻案・軽量化。

## 機械検証

### ネットリスト・導通

未結線は**2 層で別物**として両方確認する:

- [ ] **回路図レベル**: 浮きピン・未接続ネットがない。.tel エクスポート +
      grep で確認（実手順は easyeda-pro-claude-skill の規律に従う）
- [ ] **PCB レベル**: アンルーテッド（ratsnest 残り）が **0**。EasyEDA Pro の
      DRC / 未布線リストで全件列挙する。「代表的な数本を見て問題ない」は
      不可 — 意図的な未結線は理由を記録する
- [ ] 電源系: 全ての電源ピン・GND ピンが**実際の銅（配線・ベタ）で**目的の
      ネットワークに到達している。同一ネット名・zone が重なって見える・
      pad open ゼロ、はいずれも導通の証明ではない。確認手順:
      1. 铺铜（copper pour）を**再構築してから**判定する（古い铺铜形状での
         判定は無効。再構築は GUI 操作 — easyeda-pro-claude-skill 参照）
      2. 電源・GND ネットを**ネットハイライト**して各層を 1 周し、
         スリットで分断されたアイランド（どこにも繋がっていない銅）がないか見る
      3. DRC の孤立銅（isolated copper）検出が有効なら併用する

### DRC

- [ ] 実 DRC エラーが **0**。ルール値を緩めて green にしたものは PASS ではない
      （DRC ルールは発注先の実制約 = JLCPCB capability に合わせる）
- [ ] 警告は全件に目を通し、握り潰した項目は理由を記録する

### リビジョン一貫性

- [ ] 回路図・PCB・ガーバー・BOM・CPL が同一の保存状態から生成されている
- [ ] 回路図を直した後に PCB へ反映（Update PCB）済みか。BOM/CPL は最終
      レイアウトから再エクスポートしたか

### 製造制約・コスト

- [ ] 最小トレース/クリアランス・ビア径・ドリル径が JLCPCB の標準工程内。
      標準を割る feature は追加費用の原因 — 見積画面の内訳で確認する。
      代表値の目安（**正は発注画面の警告と見積内訳**。JLCPCB の
      capabilities ページで最新値を確認できる）:
      - 2 層 min trace/space: 0.127mm (5/5mil)
      - ビアドリル: 0.3mm 以上が標準。それ未満（0.15〜0.25mm）は追加費用
      - 板厚 1.6mm・外層銅 1oz・HASL が標準。ENIG・厚銅・特殊板厚は追加費用
- [ ] DRC ルール値がこの capability に合っている（緩めて green にしたルールは
      PASS ではない）
- [ ] **発注フォームの値が設計と一致**: 層数・板厚・銅厚・表面処理・最小穴径。
      フォームは設計データから自動では埋まらない — 4 層設計を 2 層で発注、
      板厚違いで嵌合部品が合わない等は全チェックを通り抜けて実物で発覚する
- [ ] 基板外形・スロット・面付けの指定が意図通り。取付穴は**めっき有無
      （PTH/NPTH）**まで指定どおりか
- [ ] Extended 部品の点数（1 部品種ごとに段取り費がかかる）が想定内

## 目視検証

EasyEDA Pro の 2D（各層個別）と 3D 表示で行う。レンダは補助であって
ゲートではない — 判定は個別項目の確認で下す。

### フットプリント ↔ 実部品の整合（発注失敗の最頻類型）

ネットリスト・DRC・実装プレビューのすべてが「正しく見える」まま通り抜ける
ため、独立した確認が必須:

- [ ] 主要部品（コネクタ・モジュール・IC・液晶）の**フットプリント寸法**を、
      発注する C 番号のデータシートのパッケージ図と突き合わせる
      （パッドピッチ・パッド幅・外形）。ジェネリック名のフットプリント
      （"SOP-8" 等）はピッチ違いの別品種が混ざりやすい
- [ ] BOM の各行: C 番号が指すパッケージと基板上のフットプリントが一致
      （0402/0603 の取り違え、同型番の電圧・容量違い）
- [ ] BOM の**値そのもの**を回路意図と照合する（10kΩ のつもりが 1kΩ は
      すべての機械検証を通過する）

### 極性・向き（実物まで潜伏する系の主戦場）

- [ ] **FFC/FPC コネクタ**: 接点面（上接点/下接点）・実装面・フリップの有無を、
      相手側（ディスプレイ等のフレックス）のデータシートと突き合わせる
- [ ] 極性部品（ダイオード・LED・電解/タンタル・コネクタ）のピン 1 /
      カソードマークがフットプリントとシルクで一致
- [ ] モジュール（XIAO 等）の向き: USB コネクタ・アンテナの向きが筐体・
      使用姿勢と整合
- [ ] 裏面実装部品のミラーリング: 裏面から見て正しい配置か

### シルク・組み立て情報

- [ ] ピン 1 マーク・極性マークが部品に隠れず読める
- [ ] コネクタの用途表記（電源/信号の区別）があると実配線ミスを防げる
- [ ] 基板名・リビジョン表記がある（複数リビジョンの実物取り違え防止）

### 機構

- [ ] 取付穴の位置・径、部品の高さ制限（ケースがある場合）
- [ ] コネクタの挿抜方向に障害物がない。基板端からのオーバーハングが意図通り

## FW 整合（PCBA でファームウェアも作る場合）

- [ ] 回路図のネット名と FW のピンマップが一致している。**両方とも単独では
      「正しく見える」**ため、突き合わせ表を作って照合する
      （ボタン・LED 等の GPIO 直結物が典型）
