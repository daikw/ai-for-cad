# JLCPCB 実装プレビュー照合（CPL レビュー簡略版）

Adapted from [Keitark/pcba-design-skills](https://github.com/Keitark/pcba-design-skills)
(MIT) `operate-jlcpcb-order/references/cpl-placement-review.md` —
プロトタイプ数量向けに簡略化。原典は量産・高価基板向けにより厳密な
機械照合（SHA-256 バインド・全行 CSV 照合スクリプト）を定義している。

実装プレビューへの到達: このプレビューは素の PCB 発注では表示されない。
PCBA（SMT Assembly）フローでガーバー→ BOM → CPL の順にアップロードした後の
部品確認ステップで初めて出る。

JLCPCB のアップロード後実装プレビューは「設計の第二の解釈」であり、
アップロードが通ったことは配置が正しいことを意味しない。CPL の回転規約・
原点規約はフットプリントごとに EDA と JLCPCB で食い違うことがある。

## アンカー校正（最初に全体ズレを検出する）

実装面ごとに、**離れた位置にある非対称・極性付きの部品を 3 点**選ぶ
（例: モジュール、コネクタ、IC。同一直線上に並ばないこと）。この 3 点で:

- [ ] X/Y が盤面上の期待位置と一致（全体の平行移動・軸入れ替え・ミラー検出）
- [ ] 回転が正しい（ピン 1 / コネクタ開口の向きで判断。パッケージ外形の
      対称性では判断しない）
- [ ] 表裏が正しい

3 点のどれかがズレていたら全体規約の問題。**ブラウザ上で直さず**、
EasyEDA Pro 側の CPL エクスポート（回転オフセット・原点）を修正して
再アップロードする。

## 全部品の目視

アンカーが合ったら、プレビューを拡大して**全部品**を確認する
（怪しい IC だけ見るのでは不足）:

- [ ] 各リード・端子がパッドに重なっている（ズレ・浮きがない）
- [ ] ピン 1・極性（ダイオード・LED・電解）の向き
- [ ] コネクタの開口方向、モジュールのアンテナ向き
- [ ] DNP 指定部品がプレビューに**載っていない**こと（手持ち在庫を使う部品が
      誤って PCBA 対象になっていると二重実装・費用増）
- [ ] 部品が基板外・他部品の上に載っていない
