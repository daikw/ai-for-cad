# 回路監査チェックリスト

Adapted from [Keitark/pcba-design-skills](https://github.com/Keitark/pcba-design-skills)
(MIT) `design-and-review-circuit/references/circuit-audit-checklist.md` —
EasyEDA Pro + JLCPCB 高速プロトタイピング向けに翻案・軽量化。

モジュールベースのデジタル基板なら「電源」「デジタル・インターフェース」
「コネクタ」の 3 節が主戦場。アナログ節は該当ブロックがある場合のみ。

## 全体構造と状態

- [ ] システム境界: 電源の入口、外部コネクタ、機能ブロック、データの流れを説明できる
- [ ] 動作状態: 起動 / 通常 / スリープ / 書き込み / 電源部分喪失 / **FW 未起動**の
      各状態で、各ピンとバスがどうなるか（HW デフォルトが安全側か）
- [ ] 共有バス（I2C/SPI）の所有権: アドレス衝突、CS の排他、マルチマスタの有無
- [ ] 回路図・ネットリスト・BOM・FW ピンマップ（あれば）の整合。
      **回路図とFWで別々に「正しく見える」ピン割当の不一致**は実物まで潜伏する

## 電源

- [ ] 電源ツリー: 入口→各レールの系統図が書ける。レール別に消費電流の概算予算
- [ ] レギュレータ: ドロップアウト余裕、安定性に必要な出力容量（データシート指定値）、
      熱（最大負荷時の損失と許容温度）
- [ ] デカップリング: 全 IC の電源ピンに対する値と配置意図。バルク容量は入口と負荷変動点
- [ ] 逆流・突入: USB とバッテリの OR 接続、ホットプラグ、逆挿し保護の要否
- [ ] モジュール内蔵電源を流用する場合: モジュールの供給能力（定格出力電流）が
      追加負荷を賄えるか

## デジタル・インターフェース

- [ ] 絶対最大定格と論理レベル: VIH/VIL・VOH/VOL の余裕、3.3V/5V 混在の変換有無
- [ ] 方向・イネーブル極性、プルアップ/ダウンの要否と値、ブートストラップピン
      （ESP32 系の GPIO0/45/46 等）への負荷の影響
- [ ] 未使用ピン: 浮かせてよいか、固定が必要か（データシートで確認）
- [ ] I2C: プルアップの有無と値、アドレスの一覧と衝突確認
- [ ] SPI/PDM/I2S 等: MCU 側のペリフェラル制約（ch 数上限・排他）と本数が合うか
- [ ] クロック・タイミングが必要なブロック: 水晶の負荷容量、配線制約は
      `[SPEC]` としてレイアウトへ引き継ぐ

## コネクタ・外部インターフェース

- [ ] 全コネクタのピン配置を相手側（ケーブル・ディスプレイ・モジュール）の
      データシートと突き合わせる。**FFC/FPC は接点面（上/下接点）と
      フリップ有無を必ず確認**（known-pitfalls の常連）
- [ ] 極性のある部品・コネクタ: 逆挿し時に何が起きるか
- [ ] 外に出る信号の保護: ESD・過電圧の要否（プロトタイプでは省略可、
      省略するなら info として記録）
- [ ] スピーカー・LED 等の駆動: MCU ピン直結の電流がドライバ要否の境界を
      超えていないか

## アナログ（該当ブロックのみ）

- [ ] 入力レンジ・バイアス・フィルタ・ADC 駆動インピーダンス
- [ ] センサーの基準（GND/リファレンス）品質とノイズ源からの分離
- [ ] スイッチング電源・モータードライバーがある場合は高速レーン外
      （人間レビュー必須のタグを付ける）

## 部品照合

- [ ] BOM の各 C 番号が回路図のシンボル・想定機能と一致（同名の別パッケージ・
      別電圧品の取り違えがないか）
- [ ] 極性部品（ダイオード・電解/タンタル・LED）の向きがシンボルと
      フットプリントで一致

## 所見の書式

各所見は「参照（refdes/ネット名）/ severity / 観測事実 / 根拠（データシート
ページ・ネットリスト行）/ 影響 / 推奨対応」。結論を支えられない項目は
`UNKNOWN` とし、必要な次の証拠（実測・データシート箇所）を書く。
