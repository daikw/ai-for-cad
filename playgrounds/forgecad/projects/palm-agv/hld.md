# Palm AGV — High-Level Design

手のひらサイズ（footprint 85〜90 mm 角、上限 100×100 mm）・全備重量 300 g 未満の屋内用 AGV
（自律走行搬送車）。卓上デモ・小物搬送・教材・検査用センサ台車を想定した差動二輪のマイクロ機。

初版ドラフトは ChatGPT (Pro) との構想設計セッション（2026-07-06）を元に、
このリポジトリの HLD 形式へ再構成したもの。[palm-uav](../palm-uav/hld.md) の姉妹プロジェクト。

## Problem

### 要求

| 項目 | 要求 |
|---|---|
| 外形寸法 | 85×85 mm 目標、最大 100×100 mm 以下。高さ 35〜55 mm |
| 全備重量 | 250 g 目標、上限 300 g（ペイロード込みで定義） |
| ペイロード | 標準 30〜50 g、最大 80 g 程度 |
| 速度 | 常用 0.10〜0.18 m/s、最高 0.25 m/s、加速度 0.3〜0.5 m/s² |
| 旋回 | その場旋回可能 |
| 稼働時間 | 30〜60 分目標 |
| 走行環境 | 屋内平滑床。勾配 3° 以下・段差 1〜2 mm 以下。カーペット・目地・ケーブル越えは対象外 |
| 位置決め精度 | ライン停止点で ±10〜20 mm。デッドレコニング単独は数 % 誤差を許容 |
| 用途 | 卓上デモ、小物搬送、教材、棚間デモ、検査用センサ台車。実工場用途（安全規格・信頼性設計）は対象外 |
| 安全 | 非常停止スイッチ、速度制限、前方障害物停止、低電圧停止、LiPo 保護 |

### 要求の優先順位

300 g 未満では、メカナムや本格 SLAM のような機能欲張りよりも**軽量・低消費電力・再現性**を
優先した方が成立性が高い。初号機は本体自重 180〜220 g + ペイロード 50 g = 総重量 230〜270 g に
置き、300 g への余裕を確保する。

1. 全備重量と重心（低重心・駆動輪荷重の確保）
2. 決められた経路を安定して走ること（再現性）
3. 安全停止（障害物・低電圧・ストール）
4. 稼働時間
5. ペイロード
6. 経路の自由度（マーカ・タグ・地図系は後段）

### このリポジトリでのスコープ

ForgeCAD でモデリングするのはシャーシ構造：上下 2 段シャーシ、モーターマウント、車輪/キャスター
取り合い、バッテリーベイ、センサマウント（ラインセンサ高さ調整含む）、ペイロード台、カバー。
モーター・基板・センサ等の電装は市販品を採寸してインターフェース設計のみ行う。
ファームウェア（速度制御・ライン追従ロジック）は CAD スコープ外。

## Approach

**差動二輪＋ボールキャスターの 2 段シャーシ構成**。下段にモーター・バッテリー、上段に制御基板・
センサ・ペイロード台を配置し、重心を低く車軸近傍に寄せる。ナビゲーションはライン追従を主とし、
エンコーダ＋IMU のデッドレコニングと床マーカ補正を組み合わせる。SLAM は載せない。

```
上面視                              側面視
┌──────────────────┐               ┌─ ペイロード台 ─┐
│ [ラインセンサ]     │← 前端下向き    │  MCU / IMU    │← 上段
│ [ToF]  [バンパー] │               ├───────────────┤
│ ┌──┐ ┌────┐ ┌──┐ │               │ モータ│LiPo│   │← 下段
│ │M │ │LiPo│ │M │ │← 車軸ほぼ中央   ══◎═══════════○══
│ └──┘ └────┘ └──┘ │                 ↑32mm車輪   ↑キャスター
│    (キャスター)    │                 最低地上高 3〜5mm
└──────────────────┘
```

### サブシステム構成

| サブシステム | 採用 | 備考 |
|---|---|---|
| 駆動 | N20 級エンコーダ付きギアモータ ×2、ギア比 150:1、6V | Pololu 150:1 Micro Metal HP 級（10×12×32 mm、11 g、210 rpm、最大効率点 0.39 kg·cm） |
| 車輪 | 直径 32 mm ゴムタイヤ ×2 直結 + ボールキャスター | 6V 無負荷で 0.35 m/s → PWM 70% で常用 0.18〜0.25 m/s |
| 電源 | 1S LiPo 500 mAh（約 10.5 g）標準、長時間版 1200 mAh（約 23 g） | 5V は昇降圧レギュレータ（Pololu S9V11F5 級）、制御系 3.3 V |
| 制御 | ESP32-S3（XIAO 級）または Raspberry Pi Pico 2 W | MCU 単体で速度制御・ライン追従・障害物停止を完結 |
| モータードライバ | DRV8835 級 2ch（1.2 A/ch 連続、0.5 g） | |
| センシング | 左右エンコーダ、前方 ToF（VL53L0X 級）、ラインセンサ（QTRX 3〜5ch）、IMU（BNO085 級）、バンパースイッチ | バンパーが最終安全停止 |
| 通信 | 開発時 USB、運用時 BLE/Wi-Fi | リアルタイム制御は無線に依存させない。停止系は車上完結 |

### 性能概算

全備 0.30 kg、車輪半径 0.016 m、転がり抵抗係数 0.04、勾配 3°、加速度 0.5 m/s² の一次見積り:

- **必要駆動力**: 転がり 0.118 N + 加速 0.150 N + 登坂 0.154 N = **0.422 N**
- **必要トルク**: 0.422 × 0.016 / 2 ≒ 0.0034 N·m/輪 ≒ 0.034 kgf·cm/輪。**安全率 10 でも 0.34 kgf·cm** で、150:1 HP の最大効率点トルク 0.39 kg·cm に収まる（ストールトルクを常用値にしない）
- **速度**: 6V 無負荷 210 rpm × 32 mm 径 = 0.35 m/s。最大効率点 170 rpm で 0.285 m/s → PWM 制限で常用域を作る
- **登坂**: トルクよりタイヤ摩擦・キャスター抵抗・重心が支配的。実用要求は勾配 3° 以下・段差 1〜2 mm 以下に抑える
- **稼働時間**: 500 mAh × 3.7 V = 1.85 Wh → 放電深度 80%・DC/DC 効率 85% で 1.26 Wh。平均 1.0 W なら 75 分、1.5 W なら 50 分、高負荷・無線常用で 20〜40 分

## Key Interfaces

| インターフェース | 内容 |
|---|---|
| モーター ⇔ 下段シャーシ | N20 胴体をギアボックス側＋胴体側の 2 点支持（片持ち接着禁止）。左右の車軸中心高さ ±0.2 mm |
| 車輪 ⇔ モーター軸 | N20 出力軸（D カット 3 mm 系）にハブ圧入または止めネジ |
| キャスター ⇔ シャーシ | 駆動輪に荷重の約 70% が残る位置。接触圧過大は走行抵抗になる |
| バッテリー ⇔ ベイ | 最下段中央。29×36×4.75 mm（500 mAh）を基準に、1200 mAh（34×62×5 mm）も収まる設計とするか要判断 |
| ラインセンサ ⇔ 前端マウント | 床面から 5〜10 mm。スロット穴で高さ調整可能にする |
| ToF ⇔ 前面 | 前方 1 個標準、左右追加可能なマウント面 |
| MCU/IMU ⇔ 上段 | M2 ネジ + インサート。IMU は車体中心近傍・モーター配線から離す |
| ペイロード ⇔ 上段台 | 車軸近傍に配置。前寄り搭載は後輪（駆動輪）荷重抜け→スリップの原因 |
| 床 ⇔ 車体底面 | 最低地上高 3〜5 mm |

## Alternatives

### 駆動方式

| 案 | 説明 | トレードオフ |
|---|---|---|
| A (採用) 差動二輪＋キャスター | 2 モーターで完結 | 最軽量・最少部品・制御容易・その場旋回可。横移動不可、キャスターが床の目地・埃に影響される |

## Usage Guide

### 組み立て

1. 下段シャーシを印刷し、モーターを 2 点支持マウントに固定する ⚠️ 左右の車軸高さが ±0.2 mm を超えると直進性が崩れる。マウントに突き当て基準面を設ける
2. 車輪をモーター軸に装着し、キャスターを取り付けて 3 点接地と駆動輪荷重（約 70%）を確認する
3. バッテリーを最下段中央のベイに固定する ⚠️ 面ファスナー等の固定不良は衝突時の LiPo 損傷リスク
4. モータードライバ・MCU・IMU を上段に M2 インサートで固定し、配線する ⚠️ モーター線はツイストし、エンコーダ線・I2C 線から離す。モーター端子のノイズ対策コンデンサも検討
5. ラインセンサを前端スロットに床面 5〜10 mm で取り付ける ⚠️ 前端のねじれ剛性が不足するとセンサ高さが走行中に変動し読値が暴れる
6. ToF・バンパースイッチを前面に取り付け、上段カバーを閉じる ⚠️ 電池交換・USB 接続・センサ高さ調整はカバーを外さずにできる開口が要る

### 走行まで

1. ファームで低電圧停止・ストール検知（電流制限/PWM 遮断）・バンパー即停止を先に実装する
2. 机上（車輪浮かせ）で左右エンコーダの符号・速度制御を確認する
3. 床でライン追従を低速（0.10 m/s）から調整し、常用 0.15 m/s へ上げる
4. 停止マーカでの停止位置再現性（±10〜20 mm）を計測する
5. ペイロード 50 g を載せ、発進スリップ・停止距離の変化を確認する ⚠️ 前寄り搭載で駆動輪荷重が抜ける

### 整備・運用

1. 走行前にキャスターの毛髪・埃の巻き込みを点検する
2. LiPo は保護回路付きセルを使い、飛行後ならぬ走行後はストレージ電圧へ
3. タイヤは摩耗・埃付着でグリップが変わる。停止精度が落ちたらまず清掃する

## Concerns

1. **キャスターの走行抵抗と引っ掛かり**: 小径ボールキャスターは毛髪・埃・床目地に弱い。ボールキャスター / 低摩擦スキッド / 後方 2 点支持の比較試験が必要。シャーシ側は交換式マウントにしておく
2. **駆動輪荷重の設計**: ペイロード位置・バッテリー位置次第で駆動輪荷重が 70% を割り、発進スリップ→オドメトリ悪化の連鎖になる。ブロックアウト段階で荷重配分を計算し、`suite.budget()` で質量・重心をチェックに含める
3. **前端剛性とラインセンサ高さ**: センサ高さ 5〜10 mm が走行振動で変動すると読値が大きく変わる。前端は薄板で張り出さず、リブで下段に結合する
4. **N20 軸への横荷重**: 車輪直結構成ではクラッシュ・段差の衝撃が出力軸に直接入る。ギアボックス破損の主因。バンパーでの停止を優先し、軸受追加は重量と相談
5. **ストール破損**: 150:1 HP のストールトルク 2.4 kg·cm は理論値で、ストールさせるとギア破損し得る（メーカー明記）。電流監視またはエンコーダ停止検知での PWM 遮断が必須
6. **ToF の検出不安定**: 黒色・斜面・透明体・強外乱光で不検出になる。最終安全停止はバンパースイッチに割り当て、ToF は減速用とする
7. **稼働時間の負荷依存**: 概算 75 分（1.0 W）〜38 分（2.0 W）と負荷で 2 倍振れる。Wi-Fi 常時接続は避け、BLE/ESP-NOW 級に抑える方針だが実測要
8. **配線・ネジ類 15〜25 g の不確実性**: 重量バジェット中で最も見積り誤差が大きい項目。試作 1 号の実測で更新する
9. **未実測寸法**: モーター・基板・電池の寸法はカタログ値。採寸前は `suite.waived()` で常時可視化する
10. **バッテリーベイの二段構成**: 500 mAh（29×36 mm）と 1200 mAh（34×62 mm）は形状が大きく異なる。共用ベイにするか版で分けるかは v0 ブロックアウトで判断
