# oldham-coupling — オルダムカップリング

## コンセプト

2本の平行シャフト間でトルクを伝達し、平行ミスアライメント（既定 1mm）と微小な角度ミスアライメントを
吸収する 3 ピース構成のフレキシブルカップリング。FDM/PLA プリント想定の摺動クリアランスで設計する。

構成: 駆動ハブ（スロット付き）+ 中央ディスク（直交タング×2）+ 従動ハブ（スロット付き、駆動側と同一部品）。

## ユーザーパラメータ（mm）

| 名前 | 既定値 | 意味 |
|---|---|---|
| shaft_dia | 6.0 | 軸径 |
| outer_dia | 30.0 | カップリング外径 |
| hub_len | 15.0 | ハブ長さ |
| slot_w | 8.0 | ハブスロット幅 |
| slot_d | 5.0 | ハブスロット深さ |
| disc_t | 8.0 | 中央ディスク本体厚 |
| fit_cl | 0.25 | 摺動嵌合クリアランス（スロット幅−タング幅、FDM 想定） |
| ax_cl | 0.3 | 軸方向クリアランス（ハブ端面〜ディスク面） |
| misalign | 1.0 | 設計許容平行ミスアライメント |

従属式:
- tongue_w = slot_w − fit_cl（タング幅）
- tongue_h = slot_d − ax_cl（タング高さ。スロット底との残り隙間 = 2×ax_cl）
- center_hole = shaft_dia + 2×misalign + 2mm（ディスク中心逃げ穴）

## 配置（グローバル座標、Z=回転軸）

- ディスク本体: z ∈ [−disc_t/2, +disc_t/2]、下タング＝X方向、上タング＝Y方向
- 駆動ハブ: 上端面 z = −disc_t/2 − ax_cl、スロット＝X方向
- 従動ハブ: 下端面 z = +disc_t/2 + ax_cl（X軸180°反転＋Z軸90°回転）、スロット＝Y方向

## 運動学（検証に使う解析解）

軸A原点、軸B (e,0)。ハブA回転角 θ のとき:
- ハブB回転角 = θ（等速 1:1）
- ディスク回転角 = θ、中心位置 = e·cosθ·(cosθ, sinθ)（半径 e/2 の円軌道）

## 完了条件

1. 全パラメータがユーザーパラメータ化され、寸法が式で従属していること
2. 静的配置で干渉体積 = 0
3. 回転スイープ（θ=0〜360°を15°刻み、e=0 / 0.5 / 1.0mm）の全ステップで干渉体積 = 0
4. 角度ミスアライメント（従動ハブ 0.5° 傾け）で干渉体積 = 0
5. ジョイント（revolute×2 + slider×1）が定義され、GUI モーションスタディを作成できる状態
6. STEP/STL エクスポート後の数値チェックで主要寸法が ±0.01mm 以内
7. 物理検証フラグ: 摺動嵌合（fit_cl=0.25mm）は実プリント・組立での確認が必要

### ジョイント構成の設計判断

- タングとスロットの拘束（スライド）は面接触機構であり、ジョイントのみでは完全表現できない。
  disc↔hub の slider ジョイントは as-built 相対値が disc_t/hub_len 変更に追従しない過拘束問題を
  起こすため採用せず、**revolute×3（各部品↔root、取付点は式駆動）** とした。
- **Motion Link 2本を定義済み**（`Link_Drive_Disc`, `Link_Drive_Driven`、いずれも 360°:360° の 1:1）。
  駆動ハブのジョイント `Rev_HubDrive` を回すだけでディスク・従動ハブが連動する
  （API 実測: J1=30°→J2/J3=30°、駆動のみ回す24ステップ全周スイープで干渉 0）。
  GUI ではブラウザのジョイントから「Drive Joints」またはアニメーションで回せる。
- 平行・角度ミスアライメント時の挙動は解析運動学（ディスク中心 = e·cosθ·(cosθ, sinθ)）に
  基づく transform スイープで数値検証済み（上表 #3, #4）。

### 簡易強度目安（PLA、参考手計算）

- タング面圧律速: 有効接触 4.4mm × 10mm × 2面 = 88mm²、許容面圧 10MPa、有効半径 10mm → 許容トルク ≈ 8.8 N·m
- NEMA17 級ステッピングモータ（〜0.5 N·m）に対し十分な余裕。高トルク用途は実機検証必須。

## ミスアライメント可動デモ（2回目のイテレーションで追加）

- **シャフト2本を追加**（Shaft コンポーネント共有、φ=shaft_dia、Dカット付き、長さ shaft_len=40mm）。
  各ハブのスロット面中心点に rigid ジョイント（オフセット式 `slot_d + 0.7 mm` / `shaft_len + slot_d + 0.7 mm`）で
  固定し、ハブと共回転する。ジオメトリベースなので **misalign 変更でシャフト位置も自動追従**する
  （as-built rigid はワールド絶対位置を焼き込み追従しないことを実測、ジオメトリベースに張り替え済み）。
- **ハブ軸穴に挿入クリアランス `bore_cl`（0.15mm）を追加**（穴=shaft_dia+bore_cl）。FDM で実際に軸が入る寸法。
- **従動軸を `misalign` パラメータ駆動の取付点にオフセット**（DrivenJointSketch 上の式駆動スケッチ点、既定 x=+1mm）。
- 最終ジョイント構成:
  - `Rev_HubDrive`（revolute、root、x=0）/ `Rev_HubDriven`（revolute、root、x=misalign、flip+整列角90°）
  - `Link_Drive_Driven`（Motion Link 1:1）→ 駆動を回すと従動が回る
  - `Slide_Disc_on_DriveSlot`（as-built slider、disc↔駆動ハブ、X方向）→ ディスクの首振り自由度
  - `Rigid_ShaftDrive` / `Rigid_ShaftDriven`（as-built rigid）
- **デモ再生**: `demo_rotation.py` を Fusion MCP の script で実行すると、e=1mm オフセット状態で
  2回転のアニメーションが再生される（ディスクのスライドは運動学解 slide = misalign·(cosθ−1) で駆動）。
- 検証: e=1mm・24角度スイープで5ボディ干渉 0。ディスク中心軌道が理論解 e·cosθ·(cosθ, sinθ) と
  一致（θ=45°: (0.5,0.5) / θ=90°: (0,0) / θ=180°: (1,0) すべて実測一致）。
  shaft.stl 体積誤差 −0.22%、hub 穴半径実測 3.0750mm（=(shaft_dia+bore_cl)/2）。
- **e=5mm でも動作検証済み**（2026-07-07）: misalign=5mm に変更 → 従動軸・従動シャフト・
  ディスク逃げ穴（φ18）がすべて自動追従し、24角度スイープで5ボディ干渉 0。
  ディスクのスライド一般式は slide = misalign·cosθ − E_REF（E_REF=1mm はスライダー作成時基準、
  demo_rotation.py に定数として記載）。**現在ドキュメントは misalign=5mm の状態のまま**。

### 既知の制限（実測に基づく）

- 閉ループジョイント（slider 2本で両ハブとディスクを結ぶ構成）は Fusion の API からは駆動できない
  （revolute の rotationValue セットが no-op になる）。このため上ハブ↔ディスク間の slider は張らず、
  幾何整合はディスクのスライド値を運動学解でセットすることで担保している（干渉チェックが監視）。
- GUI で駆動ハブを**アセンブリドラッグ**するとディスクのスライドは追従しないので、タングが
  スロット側面に見かけ上食い込むことがある（拘束が接触を知らないため）。正確な連動は
  `demo_rotation.py` か、Contact Sets（接触セット）有効化で確認する。
